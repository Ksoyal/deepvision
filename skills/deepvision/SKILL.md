# DeepVision

使用 DeepVision 将图片解析为层级视觉表示：先读取语义单元
（composites），再按需引用带坐标的视觉基元（primitives）。

## 是否使用

- 当前模型无法直接看图，或图片未进入模型上下文：使用。
- 用户要求查看剪贴板、刚截的图或剪贴板里的内容：使用。
- 任务需要 OCR、坐标、空间关系、稳定 ID 或结构化抽取：使用，即使当前模型能直接看图。
- 当前模型能直接看图，且用户只需要普通、非结构化的图片理解：可以直接回答，无需重复调用。
- 当前对话已有同一图片的有效 DeepVision 解析结果：直接复用。

## 执行流程

1. 根据图片来源选择命令：
   - IDE、聊天工具或用户提供了明确图片路径、URL 或 data URI：直接运行 `deepvision <输入>`。即使图片来自粘贴或截图，也优先使用环境已提供的明确路径。
   - 用户只提到剪贴板、刚截图或剪贴板里的图片，且没有明确路径：直接运行 `deepvision -c`。
   - 用户要求程序化结构：添加 `--json`。
2. 如果当前对话已经包含同一图片的完整、有效 DeepVision 解析结果，直接复用；除非用户要求重新识别、图片已变化或现有结果不足，不要重复调用。
3. 基于解析结果完成用户原始任务，例如解题、抽取表格、解释界面或定位控件；不要只复述或原样倾倒工具输出。
4. 优先使用 `# 语义单元` 中的 `role`、`text` 和 `children` 理解整体结构。需要精确位置或核对细节时，再读取 `# 视觉基元` 中的 `parent`、`bbox`、`point` 和 `text`。
5. 使用关系辅助空间判断。语义关系来自视觉模型；几何关系由坐标推导，并限制在同一父语义单元内。

## 纯剪贴板规则

- 当用户提到剪贴板或刚截图，且环境没有提供明确图片路径时，必须先运行 `deepvision -c`。
- 不要调用 PowerShell、Python 或其他工具把剪贴板图片保存到临时文件。
- 不要在运行 `deepvision -c` 前自行探测或转换剪贴板内容。
- 如果 IDE 已提供 `[Image: <路径>]` 或等价的明确路径，按该路径解析，不适用本节。
- 如果 `deepvision -c` 明确报告剪贴板没有图片，告知用户重新复制图片，或请用户提供现有图片路径。不要自行创建临时文件作为降级方案。

## 输入与路径

- 使用环境或用户提供的原始路径、URL 或 data URI。
- 不要猜测、拼接、搜索或自行替换图片路径。
- 明确路径不存在或不可访问时，报告该输入无效并请用户提供有效输入。
- 不要仅因文件位于临时目录就拒绝使用；环境已提供的临时路径属于明确图片输入，不等同于 agent 自行把剪贴板图片落盘。

## 核查与缓存

- 默认使用缓存，不要主动添加 `--no-cache`。
- 优先复用当前对话中已有的同图解析结果，其次使用 DeepVision 默认缓存。
- 仅在用户明确要求重新识别、图片内容已更新但结果疑似仍来自旧缓存、现有解析不足，或正在调试 prompt/模型时添加 `--no-cache`。
- 解析结果用于自动点击、关键数据提取等高风险操作时，使用 `--verify`。
- 需要逐个核查所有基元时才使用 `--verify full`；该模式请求多、速度慢。

## 错误处理

- 命令不存在：提示安装 DeepVision，并确认命令在当前环境的 `PATH` 中。
- 配置缺失：提示配置 `DEEPVISION_API_KEY`、`DEEPVISION_BASE_URL`、`DEEPVISION_MODEL`，或运行 `deepvision init`。
- 剪贴板无图片：按纯剪贴板规则处理，不绕过 `deepvision -c`。
